## 🏗️ 1. Các Nguyên Lý Cơ Bản (Coding Principles)

| Nguyên lý | Ý nghĩa Chuyên môn | Giải thích Liên tưởng & Cấp độ Ứng dụng |
|---|---|---|
| **DRY** (Don't Repeat Yourself) | Khử sự trùng lặp (duplication) trong code logic. | **Liên tưởng:** Nếu copy-paste một đoạn code 2 lần, lúc sửa lỗi phải ráp mảng sửa lại 2 chỗ.<br/>**Ứng dụng:** Gom nhóm các chức năng định dạng Date, tính Thuế vào 1 file `utils` dùng chung. |
| **KISS** (Keep It Simple, Stupid) | Tối giản hóa giải pháp thuật toán và thiết kế hệ thống. | **Liên tưởng:** Thay vì dùng dao mổ trâu (Regex dài dòng) để giết gà, hãy dùng cấu trúc cơ bản `if/else`.<br/>**Ứng dụng:** Viết code sao cho người mới vào team đọc cũng hiểu ngay. |
| **YAGNI** (You Aren't Gonna Need It) | Tránh hiện tượng Over-engineering (thiết kế dư thừa). | **Liên tưởng:** Mua áo khoác mùa đông trước 5 năm vì định cư sang Mỹ, nhưng rốt cuộc không đi.<br/>**Ứng dụng:** Sprint có gì code cái nấy, không làm dư API "phòng hờ sau này khách hàng đòi". |
| **SoC** (Separation of Concerns) | Phân tách ranh giới của các modules dựa trên mục tiêu xử lý. | **Liên tưởng:** Nhân viên phục vụ không được chui vào bếp nấu ăn.<br/>**Ứng dụng:** Tách rạch ròi file Giao diện (UI) và file chứa logic Tính Toán/Gọi API (Services). |
| **CoI** (Composition over Inheritance) | Ưu tiên quan hệ "Has-a" (Sở hữu) hơn là "Is-a" (Kế thừa). | **Liên tưởng:** Lắp ghép Lego dễ biến đổi hơn là đúc 1 cục nhựa nguyên khối.<br/>**Ứng dụng:** Lắp ghép các interface/mảnh chức năng lại thay vì chuỗi kế thừa `A extends B extends C...` dài bất tận. |
| **LoD** (Law of Demeter) | Giảm thiểu mức độ phụ thuộc (Coupling) bằng cách giới hạn sự giao tiếp nội bộ giữa các đối tượng. | **Liên tưởng:** Muốn mượn tiền sếp thì xin sếp, đừng lục ví sếp để tự lấy thẻ ATM.<br/>**Ứng dụng:** Chặn không gọi dạng chuỗi sâu kiểu `user.getAccount().getBank().process()`. |
| **Fail Fast** | Lập tức báo lỗi (Throw Exception) tại điểm bắt đầu xảy ra sai lệch trạng thái. | **Liên tưởng:** Thấy rác ở cửa thì vứt ngay, đừng đem giấu vào máy giặt.<br/>**Ứng dụng:** Kéo giãn Validate Input ở đầu hàm, nếu sai quăng Exception lập tức thay vì return `0` cho chạy mù. |
| **CoC** (Convention over Configuration) | Giảm tải các quy định cấu hình khai báo bằng các Quy ước Mặc định. | **Liên tưởng:** Đồng phục công sở, ai cũng mặc áo trắng quần đen, đỡ tốn công suy nghĩ phối màu.<br/>**Ứng dụng:** Trong Next.js cứ đặt file `page.js` vào folder là auto thành Route, không cần config ruột rà. |

## 💎 2. SOLID Principles (Nguyên Lý OOP)

| Nguyên lý | Ý nghĩa Chuyên môn | Giải thích Liên tưởng & Cấp độ Ứng dụng |
|---|---|---|
| **[S] SRP** (Single Responsibility) | Một class/module chỉ đảm nhận duy nhất một trách nhiệm (1 lý do để thay đổi). | **Liên tưởng:** Một cái kềm thì không nên tích hợp thêm tính năng sạc pin điện thoại.<br/>**Ứng dụng:** Class `User` không được chứa hàm `saveToDatabase` hay `sendEmail`. |
| **[O] OCP** (Open/Closed) | Thực thể mở cho việc mở rộng (extension), nhưng đóng cho việc sửa đổi (modification). | **Liên tưởng:** Muốn sút bóng mạnh hơn hãy mua giày đinh, đừng phẫu thuật cấy đinh vào chân.<br/>**Ứng dụng:** Thay vì dùng `if-else`/`switch-case` khổng lồ, dùng Interface (Polymorphism) để triển khai class mới. |
| **[L] LSP** (Liskov Substitution) | Đối tượng của class cha phải được thay thế an toàn bằng đối tượng class con mà không gây lỗi hành vi. | **Liên tưởng:** Chim cánh cụt (con) kế thừa loài Chim (cha). Chim cha biết bay, ép Cánh cụt bay sẽ gây sụp đổ hệ sinh thái.<br/>**Ứng dụng:** Phân mảng lại thiết kế ban đầu bằng các Interface hẹp hơn (`Flyable`, `Swimmable`). |
| **[I] ISP** (Interface Segregation) | Tránh tạo ra các Interface phình to (Fat Interface). Client không bị ép phụ thuộc method chúng không cần. | **Liên tưởng:** Hợp đồng thuê nhà không được ép người thuê phải trả chi phí thức ăn cho thú cưng nếu họ không nuôi chó.<br/>**Ứng dụng:** Tách riêng `IPrinter` và `IScanner` thay vì gộp chung làm 1 cái `IMachine`. |
| **[D] DIP** (Dependency Inversion) | Logic tầng cao và hạ tầng tầng thấp đều phải phụ thuộc vào các Abstraction (Interface). | **Liên tưởng:** Rút dây cắm điện chứ không hàn chặt dây máy quạt thẳng vào mạng điện tường nhà.<br/>**Ứng dụng:** Truyền Database thông qua tham số khởi tạo `(Dependency Injection)` thay vì code chết `new MySQL()` trong Controller. |

## 🧩 3. Design Patterns (Trọn bộ 23 Mẫu GoF)

### 3.1 Nhóm Creational (Khởi tạo Đối tượng)

| Pattern | Ý nghĩa Chuyên môn | Giải thích Liên tưởng & Cấp độ Ứng dụng |
|---|---|---|
| **Singleton** | Đảm bảo 1 class chỉ có đúng 1 instance duy nhất được cấp phát. | **Liên tưởng:** Một quốc gia chỉ có một Vị Vua.<br/>**Ứng dụng:** Hồ chứa Database Connection Pool, Logger, Caching. |
| **Factory Method** | Giao quyền quyết định khởi tạo object cho các class con thông qua hàm/Interface chung. | **Liên tưởng:** Đặt xưởng sx làm Áo, đưa mẫu nào xưởng ra mẫu đó mà khách không cần tự may.<br/>**Ứng dụng:** Tạo tài liệu Word, PDF tuỳ Input truyền vào hàm `create()`. |
| **Abstract Factory** | Tạo ra tập hợp các "Họ" (families) object liên quan tới nhau thay vì tạo từng object riêng rẽ. | **Liên tưởng:** Xưởng mộc tạo ra Nguyên Bộ Bàn Ghế (Phong cách Âu/Á) đồng bộ nhau.<br/>**Ứng dụng:** Chế tạo UI đồng bộ đa nền tảng (`WinButton, WinTextbox` vs `MacButton, MacTextbox`). |
| **Builder** | Bóc tách quá trình khởi tạo một object nội dung cực kì phức tạp ra khỏi biểu diễn. Xây step-by-step. | **Liên tưởng:** Cấu hình 1 chiếc Pizza (thêm phô mai, thêm xúc xích, đổi đế dày...).<br/>**Ứng dụng:** Cấu hình tạo URL Builder, hoặc Entity nhiều fields (SQL Query Builder). |
| **Prototype** | Clone (Nhân bản) một object hiện có thay vì tạo mới đồ thị từ đầu để tiết kiệm Memory. | **Liên tưởng:** Dùng máy in màu copy tranh sẽ rẻ và nhanh hơn thuê họa sĩ vẽ lại từ đầu.<br/>**Ứng dụng:** Nhân bản lính quái vật hàng loạt trong Game liên tục. |

### 3.2 Nhóm Structural (Cấu trúc Lắp ráp)

| Pattern | Ý nghĩa Chuyên môn | Giải thích Liên tưởng & Cấp độ Ứng dụng |
|---|---|---|
| **Adapter** | Chuyển đổi (Wrapper) Interface class này thành Interface mà class khác đang cần sử dụng. | **Liên tưởng:** Phích cắm 3 chấu gắn vào chấu chuyển đổi để cắm vô được lỗ điện 2 chấu.<br/>**Ứng dụng:** Chuyển đổi mã XML lộn xộn từ API cũ sang JSON mượt mà cho React Hook. |
| **Decorator** | Đắp thêm chức năng vào 1 object vào thời gian chạy (Runtime) mà không can thiệp vào Class đó. | **Liên tưởng:** Mặc giáp xịn hay đeo kiếm xịn cho Hero sẽ làm tăng Stat của Hero lập tức.<br/>**Ứng dụng:** Thêm tính năng Logging/Encryption I/O Stream (`BufferedInputStream`). |
| **Facade** | Cung cấp 1 Interface siêu lớp đại diện tối giản, che khuất hệ thống Sub-system bề bộn bên dưới. | **Liên tưởng:** 1 Nút "Bấm Đặt Xong" trên Grab che giấu quá trình định vị GPS, Check Ví Momo, Kiếm Bác Tài.<br/>**Ứng dụng:** Viết 1 SDK `App.start()` để chạy ngầm Database, Firewall, Background Jobs. |
| **Proxy** | Cung cấp vật thay thế (Placeholder) để kiểm soát gắt gao đường truy cập vào class Thật sự phía sau. | **Liên tưởng:** Thẻ từ cửa quét vân tay công ty, chỉ được vào nếu gõ đúng mã PIN (hoặc bảo vệ gác cổng).<br/>**Ứng dụng:** Lazy Loading Hibernate (Chỉ load dữ liệu khi cần), chặn chặn Firewall, Data Cache. |
| **Bridge** | Tách rời thành phần Trừu tượng (Abstraction) và Thực thi (Implementation) để chúng tiến hóa độc lập. | **Liên tưởng:** Cục Điều khiển Remote Cầm tay và Các dòng Smart TV. Hai cái pt song song độc lập.<br/>**Ứng dụng:** Code 1 Driver màn hình nhưng tương thích chạy tốt trên Windows, Linux, MacOS. |
| **Composite** | Hợp nhất các object thành mô hình cây (Tree) và xử lý một tập hợp đối tượng hệt như 1 đối tượng duy nhất. | **Liên tưởng:** Trút 1 Giỏ To (trong Giỏ chứa trái táo và Giỏ con). Bất kể Thùng to/nhỏ thì hành động trút giống y nhau.<br/>**Ứng dụng:** Hệ thống duyệt Folder Cây Thư Mục, UI Components Tree trong React (Element bọc Element). |
| **Flyweight** | Chia sẻ các cụm thuộc tính dùng chung để giảm tới đáy việc sử dụng RAM cho hàng triệu phần tử. | **Liên tưởng:** Nhìn vào rừng có 1 triệu cái cây. Thực chất System chỉ lưu bộ vân cấu tạo "Lá Cây" ĐÚNG 1 LẦN chia chung cho toàn rừng, chỉ lưu toạ độ riêng biệt 1 triệu vị trí.<br/>**Ứng dụng:** Render hệ thống game 3D thế giới mở tốn ít bộ nhớ. |

### 3.3 Nhóm Behavioral (Hành vi & Tương tác)

| Pattern | Ý nghĩa Chuyên môn | Giải thích Liên tưởng & Cấp độ Ứng dụng |
|---|---|---|
| **Observer** | Mô hình Đăng ký / Thông báo (Pub-Sub) Một chiều. Một Subject thay đổi, toàn bộ Observers bị cập nhật. | **Liên tưởng:** YouTube Channel up video mới -> 1 triệu chuông điện thoại Subscriber reo lên cùng lúc.<br/>**Ứng dụng:** DOM Event Listeners (`onClick`), React Redux State, Socket.io Broadcast. |
| **Strategy** | Tách rời Bộ thuật toán thành các Object riêng, nhét vào bối cảnh lúc Runtime một cách linh động. | **Liên tưởng:** Nút "Thanh Toán". Ông ấn MoMo thì dùng `Strategy MoMo`, ấn Thẻ xài `VisaStrategy`.<br/>**Ứng dụng:** Thuật toán Nén File (áp dụng Zip hay Rar), Thuật toán Mẻ Sorting. |
| **Command** | Đóng gói một Yêu cầu thao tác (Request/Action) thành Một Đối Tượng thực thể (Object) riêng lẻ. | **Liên tưởng:** Tờ Order gọi món trong quán ăn. Tờ này sẽ truyền từ Khách -> Bồi Bàn -> Nhà Bếp.<br/>**Ứng dụng:** Hệ thống Undo/Redo (Ctrl+Z) trong MS Word, Hệ thống Command Queue xử lý Asynchronous. |
| **State** | Class sẽ hoán đổi toàn bộ Hành vi ngay lập tức khi Trạng thái Nội bộ của Object được chuyển đổi. | **Liên tưởng:** Máy bán nước tự động (Trạng thái hết nước $\to$ Trạng thái nhận tiền $\to$ Nhả lon).<br/>**Ứng dụng:** Bộ máy nhâm phẩm Trạng thái Nhân vật Game (Đang thức, Đang bị Stun, Đang chết). |
| **Template Method** | Dựng Cột Sống thuật toán khung trong Class Cha. Cho qua các bước thực thi vụn vặt cho Class Con (Override). | **Liên tưởng:** Khung làm nhà (Đổ móng $\to$ Xây tường $\to$ Lợp mái). Còn mái ngói hay lá chuối là việc của Ông thầu xây dưng (Class Con).<br/>**Ứng dụng:** Vòng đời (Lifecycles) Component React (`componentDidMount`, `componentWillUnmount`). |
| **Iterator** | Trích xuất công cụ Đọc Dữ Liệu tuần tự ra khỏi Cấu trúc lõi, duyệt danh sách không cần lộ thiết kế chìm. | **Liên tưởng:** Bộ Remote tivi có nút Bấm Tới Kênh Kế Tiếp `Next()`, bất kể Tivi xài ăng ten hay cáp chảo.<br/>**Ứng dụng:** Các vòng lặp danh sách chuẩn `for...of`, Interface Iterable trong Javascript/Java Array, Map, Set. |
| **Mediator** | Triệt tiêu mọi con giáp rối rắm của mạng nhện Class bằng 1 Trung gian Phối hợp Lệnh. | **Liên tưởng:** Đài Không Lưu Sân bay. Máy bay A không gọi điện Máy bay B cãi nhau giành lane. Tất cả báo cáo lên Đài để Đài phán quyết xếp chỗ.<br/>**Ứng dụng:** Quản lý hàng đống component Input chéo nhau trong Form phức tạp. Chat Room. |
| **Chain of Responsibility** | Ném Request vào 1 mảng Dây chuyền Các Trạm xử lý nối tiếp. Trạm 1 Pass thì văng qua Trạm 2 tiếp tục. | **Liên tưởng:** Phê duyệt mua máy tính $\rightarrow$ PM (Không đủ cấp độ) $\rightarrow$ Đẩy lên Sếp Tổng (Ký).<br/>**Ứng dụng:** Hệ thống Middlewares của NodeJS Express (`app.use(auth) -> app.use(logger) -> app.use(route)`), Lọc Spam Mail. |
| **Visitor** | Rút logic thuật toán xử lý lặp ra NHỎ KHỎI đồ thị node. Add Method xài trên toàn tập hợp Tree mà không thay đổi bất kì Node Class nào. | **Liên tưởng:** Người Giao Hàng đi vào Khu Cư xá A, Gõ Cửa các Node Nhà và Vất lại từng loại Thư tương ứng.<br/>**Ứng dụng:** Máy Parse cây DOM Trình duyện xuất ra Text ASCII, Cỗ máy Compiler đọc biến Ast Tree. |
| **Memento** | Chụp (Snapshot) và bảo lưu vẹn nguyên Trạng thái bí mật của đối tượng bên hông tại 1 thời điểm. Phục hồi mướt rượt. | **Liên tưởng:** Lệnh Quick Save Point trong game Final Fantasy, lúc chết rớt đồ thì Resume Load lại y đúc.<br/>**Ứng dụng:** Git Commits History, Save State Undo/Redo. |
| **Interpreter** | Nhận diện Ngôn Ngữ riêng rẽ (Grammar rules) biến thành Hành động chạy chuẩn Syntax Tree. | **Liên tưởng:** Phiên dịch viên Pháp -> Việt Nam.<br/>**Ứng dụng:** Biểu thức Regular Expressions (Regex), Công Cụ Viết SQL Query Language (SQL Parser). |

## 🏢 4. Architecture Patterns (Kiến Trúc Phần Mềm)

Làm thế nào để hệ thống gánh được triệu người dùng?

| Kiến Trúc | Ý nghĩa Chuyên môn | Giải thích Liên tưởng & Cấp độ Ứng dụng |
|---|---|---|
| **MVC** | Tách cấu trúc hệ thống thành Model (Data), View (UI), và Controller (Routing & Logic). | **Liên tưởng:** Nhà hàng có: Kho nguyên liệu (Model), Khách ngồi bàn (View) và Bồi bàn (Controller).<br/>**Ứng dụng:** Trùm sò trong Web: Laravel, Rails, Spring Boot. |
| **MVVM** | (Model-View-ViewModel) Data đổi $\to$ Giao diện tự động đổi theo và tự Bind lại thông qua Reactive Observer. | **Liên tưởng:** Bản nhạc phổ vạch (View-Model) -> Cứ bản nhạc chạy nốt nào, ngón tay piano múa nốt đó.<br/>**Ứng dụng:** Cốt lõi của các Framework Front-end: Vue, Angular, React, SwiftUI IOS. |
| **Clean Architecture / Hexagonal** | Phân tầng thành các Vòng tròn Lớp cô lập sự phụ thuộc lõi (Domain) khỏi Database và Frontend (Dependency Inversion). | **Liên tưởng:** Quả cầu củ hành. Lõi ở giữa là Thuyết Trọng Yếu, các lớp vỏ ngoài là Khung Sắt Bọc Chống Lão hóa.<br/>**Ứng dụng:** Viết Service lớn. Bảo vệ Core logic để đổi DB từ MySQL sang Postgres mà app không sập chênh vênh. |
| **Microservices** | Rã nhỏ hệ thống nguyên khối thành mạng lưới các Dịch vụ độc lập nhỏ lẻ liên kết với nhau bằng RestAPI hoặc Message. | **Liên tưởng:** Thay vì 1 tướng chỉ huy 10 vạn quân, giờ chia làm 10 toán độc lập, cánh quân này tử trận cánh khác vẫn đánh tốt.<br/>**Ứng dụng:** Shopee, Netflix (Service Payment sập thì Service Xem phim vẫn vận hành ngon lành, chịu tải Flash Sales bùng nổ). |
| **Monolithic** | Kiến trúc Hệ thống Nguyên Khối Truyền thống. Nhét hết 100 Modules logic, DB models, HTML View vào chung 1 File Repository. | **Liên tưởng:** Hàng cháo chửi 1 mình bà chủ vừa bưng mâm, vừa thu tiền, vừa vắt chanh múc cháo.<br/>**Ứng dụng:** Nhóm Startup Test tính năng (MVP) mới ra mắt cần Deploy nhanh. |
| **Layered / N-tier** | Mẫu kiến trúc chia tầng phân lớp chặt chẽ (Presentation, Business Logic, Data Access), các tầng chỉ được gọi tầng ngay dưới nó. | **Liên tưởng:** Xếp lớp hành chính: Công nhân gọi cho Tổ trưởng -> Tổ trưởng trình Giám đốc. Lôi ông Giám đốc hỏi chuyện quét rác là sai bét quy tắc.<br/>**Ứng dụng:** Backend Spring Boot kinh điển: `Controller -> Service -> Repository`. Tách rõ ranh giới chuyên môn cực gọn gàng. |
| **Serverless** | Thuê Cloud tự cấp phát sức mạnh để chạy một đoạn Function (Logic) mà đ*ch cần quan tâm mua máy chủ (Server), Server tự co giãn. | **Liên tưởng:** Nhà nghèo, đi ăn cỗ lấy phần thay vì bỏ vài củ xây cái bếp lửa. Khi nào nhai Mỳ thì mướn hớp Nước Sôi, có xài có trả tiền.<br/>**Ứng dụng:** Viết Cloud Functions (AWS Lambda). Web đang lay lắt 1 khách thì hóa đơn Server 0$, đang yên LiveStream giáp lá cà 1 triệu mống $\to$ Serverless bành trướng đẻ ra 1 vạn con Functions chốt đơn không sụp! |
| **Event-Driven / CQRS** | Tách hẳn Database Đọc và Database Ghi. Tách luềng Data Stream xử lý Bất Đồng Bộ bằng bắn Event Message Queues. | **Liên tưởng:** Mua vé Concert bằng Web Cổng 1 thu tiền (Ghi), xong cổng bắn Event cho Cổng In Chữ ra vé Cổng 2.<br/>**Ứng dụng:** Thắt cổ chai siêu kinh điển (Phân tách đọc viết) Message Broker Kafka ở RabbitMQ. Hệ thống Live Feed Twitter Realtime mượt mà. |

## 🚀 5. Software Engineering Practices

Lập trình viên đẳng cấp vượt trội kỹ thuật gõ code phải nắm vững quy trình này:

| Khái niệm | Ý nghĩa Chuyên môn | Giải thích Liên tưởng & Cấp độ Ứng dụng |
|---|---|---|
| **TDD** (Test-Driven) | Phát triển code dựa trên viễn cảnh xây dựng các kịch bản Test hỏng (Red Face) từ trước, Code Pass (Green Face) sau. | **Liên tưởng:** Viết sẵn đề kiểm tra kỳ 1, rồi bắt buộc đi tìm kiến thức làm sao để Giải xong cho đề chấm đỗ "Xanh".<br/>**Ứng dụng:** Viết Test đè trước, Code theo sau -> Phát hiện rò rỉ sớm, Regression 100% bug free vững chãi. |
| **BDD** (Behavior-Driven) | Viết Test tập trung vào Hành Vi người dùng, viết bằng ngôn ngữ Kinh Doanh tự nhiên (Given-When-Then) cho phép team Phi-Kỹ-Thuật cùng hiểu. | **Liên tưởng:** Viết kịch bản diễn tuồng trước khi quay thật: "GIVEN (Có mưa) THEN (Cầm dù)".<br/>**Ứng dụng:** Khách hàng khú đế của BA quăng File Kịch Bản (Gherkin/Cucumber), Automation Tester vọc luôn lệnh Chạy Test. Code sai kịch bản $\to$ Lỗi! Cực ăn xăm giữa Sếp & Dev. |
| **CI/CD** | Tự động hóa chuỗi Quy trình Tích hợp và Phân phối Release Web thẳng ra Internet trên môi trường Không ngừng. | **Liên tưởng:** Đưa nguyên liệu băng chuyền $\to$ Máy tự Cắt $\to$ Rửa $\to$ Đóng hộp mỳ xuất xưởng không cần tay đụng trực tiếp.<br/>**Ứng dụng:** Viết File Github Actions Pipeline đẩy commit là tự Auto Chạy Test -> Pass -> Auto Build Docker -> Deploy Vercel Production. |
| **12-Factor App** | 12 Quy tắc Phương pháp luận xây dựng các Hệ Ứng dụng Chạy Trực Cấp độ Cloud-Native Vĩnh Cửu. | **Liên tưởng:** Nội quy phòng thủ để con tàu vũ trụ sống sót khi chuyển bay từ Mặt Trăng sang Hỏa Tinh không nổ máy.<br/>**Ứng dụng:** Phá hủy thói quen Hardcode Mật khẩu. Mọi Setting nhạy cảm cắm ngoài Database Variable file `.env`, Logging xuất Streaming stdout. |
| **Refactoring & Code Smells** | Quy trình đục vát (Refactoring) những đoạn code Thúi (Code Smells), thay đổi ruột lằng nhằng bằng thiết kế sạch MÀ KHÔNG GÂY LỖI NGHIỆP VỤ CUỐI. | **Liên tưởng:** Rửa cái Tủ Lạnh Nhớt Thúi. Hộp thức ăn vẫn 10 cái y chang nhưng sạch bong rạch ròi, rút giò dễ lấy.<br/>**Ứng dụng:** Lôi Code dài miên man gợp 30 Method về chuẩn SRP, Gỡ "Magic Number", Chia module logic thảm họa. |
| **DDD** (Domain-Driven Design) | Bố trị cấu trúc Code và Bảng Thiết Kế Database ăn khớp hoàn hảo 100% theo Ngôn Ngữ Chuyên Ngành Kinh Doanh ngoài đời thật (Ubiquitous Language). | **Liên tưởng:** Nông dân nói "Lúa Trổ Bông". Code cũng ghi y chang `Paddy.flower()` chứ không nói `Item.updateState()`.<br/>**Ứng dụng:** Bản đồ ranh giới chia siêu chuẩn cắt rạch Microservices ko vỡ trận, 2 Team BA và Team Coder cùng 1 góc nhìn logic doanh nghiệp tỷ đô. |
