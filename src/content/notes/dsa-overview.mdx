## 🤔 Tại sao phải học DSA?

Hãy tưởng tượng bạn có **1 tỷ cuốn sách** trong thư viện. Nếu sách xếp lung tung → bạn mất **cả đời** để tìm 1 cuốn. Nhưng nếu sách được **sắp xếp theo chủ đề, theo tên** → bạn tìm trong **vài giây**.

**Cấu trúc Dữ liệu** = Cách bạn **sắp xếp** sách trong thư viện.  
**Giải thuật** = Cách bạn **tìm kiếm** cuốn sách bạn cần.
> Hiểu DSA = Viết code **nhanh hơn**, **tốn ít bộ nhớ hơn**, và **giải quyết vấn đề thông minh hơn**.

## 📐 Complexity Analysis — Phân tích Độ phức tạp

### Big O Notation /bɪɡ oʊ noʊˈteɪ.ʃən/ là gì?

**Big O** là cách đo **tốc độ** (Time Complexity /taɪm kəmˈplɛk.sɪ.ti/) và **bộ nhớ** (Space Complexity /speɪs kəmˈplɛk.sɪ.ti/) của một giải thuật khi dữ liệu **tăng lên**.

> Hãy tưởng tượng bạn tìm 1 cuốn sách trong thư viện có **n** cuốn. Big O cho biết: khi thư viện lớn gấp 10 lần, bạn mất **bao lâu hơn**?

### Bảng Complexity từ nhanh → chậm

| Big O | Tên (English) | Phát âm IPA | Ý nghĩa | Ví dụ đời thực |
|-------|---------------|-------------|---------|-----------------|
| **O(1)** | Constant /ˈkɒn.stənt/ | Hằng số | Luôn cùng tốc độ, bất kể n lớn cỡ nào | Mở ngăn kéo biết chính xác vị trí |
| **O(log n)** | Logarithmic /ˌlɒɡ.əˈrɪð.mɪk/ | Logarit | Mỗi bước loại bỏ **nửa** dữ liệu | Tìm từ trong từ điển (chia đôi) |
| **O(n)** | Linear /ˈlɪn.i.ər/ | Tuyến tính | Tăng tỉ lệ thuận với n | Đọc từng trang sách từ đầu đến cuối |
| **O(n log n)** | Linearithmic /ˌlɪn.i.əˈrɪð.mɪk/ | Log tuyến tính | Chia nhỏ + xử lý từng phần | Merge Sort, Quick Sort |
| **O(n²)** | Quadratic /kwɒˈdræt.ɪk/ | Bình phương | Vòng lặp lồng vòng lặp | So sánh từng cặp học sinh |
| **O(n³)** | Cubic /ˈkjuː.bɪk/ | Lập phương | 3 vòng lặp lồng nhau | Nhân ma trận |
| **O(2ⁿ)** | Exponential /ˌek.spəˈnen.ʃəl/ | Hàm mũ | Gấp đôi mỗi bước | Thử tất cả tổ hợp mật khẩu |
| **O(n!)** | Factorial /fækˈtɔː.ri.əl/ | Giai thừa | Tất cả hoán vị | Bài toán Người bán hàng |

### So sánh trực quan

```
n = 10        n = 100       n = 1000
O(1)      1          1            1         ← Luôn nhanh!
O(log n)  3          7           10         ← Rất nhanh
O(n)     10        100         1000         ← Ổn
O(n²)   100     10,000     1,000,000        ← Chậm dần
O(2ⁿ)  1024   1.27×10³⁰    ∞ 💀           ← Không khả thi!
```

### Thuật ngữ quan trọng

| Tiếng Việt | English | IPA | Giải thích |
|------------|---------|-----|------------|
| Độ phức tạp thời gian | Time Complexity | /taɪm kəmˈplɛk.sɪ.ti/ | Code chạy **nhanh hay chậm** |
| Độ phức tạp không gian | Space Complexity | /speɪs kəmˈplɛk.sɪ.ti/ | Code dùng **nhiều hay ít bộ nhớ** |
| Trường hợp tốt nhất | Best Case | /bɛst keɪs/ | Input lý tưởng |
| Trường hợp trung bình | Average Case | /ˈæv.ər.ɪdʒ keɪs/ | Input thông thường |
| Trường hợp xấu nhất | Worst Case | /wɜːrst keɪs/ | Input tệ nhất |
| Tại chỗ | In-place | /ɪn pleɪs/ | Không dùng thêm bộ nhớ |
| Ổn định | Stable | /ˈsteɪ.bəl/ | Giữ thứ tự phần tử bằng nhau |
| Chia để trị | Divide and Conquer | /dɪˈvaɪd ænd ˈkɒŋ.kər/ | Chia nhỏ bài toán → giải → gộp |
| Tham lam | Greedy | /ˈɡriː.di/ | Chọn tốt nhất **ngay lúc này** |
| Quy hoạch động | Dynamic Programming | /daɪˈnæm.ɪk ˈproʊ.ɡræm.ɪŋ/ | Ghi nhớ kết quả, không tính lại |
| Quay lui | Backtracking | /ˈbæk.træk.ɪŋ/ | Thử → sai → quay lại thử cách khác |

**Quy tắc**: Big O càng nhỏ → code càng nhanh. Luôn cố gắng chọn giải thuật có Big O thấp nhất!

## 📊 Bảng so sánh Time Complexity

### Data Structures

| Cấu trúc | Access | Search | Insert | Delete |
|-----------|--------|--------|--------|--------|
| Array | O(1) | O(n) | O(n) | O(n) |
| Hash Table | O(1) | O(1) | O(1) | O(1) |
| Linked List | O(n) | O(n) | O(1) | O(1) |
| Stack/Queue | O(n) | O(n) | O(1) | O(1) |
| BST (balanced) | O(log n) | O(log n) | O(log n) | O(log n) |
| Heap | O(1) top | O(n) | O(log n) | O(log n) |

### Sorting Algorithms

| Giải thuật | Best | Average | Worst | Space | Stable |
|-----------|------|---------|-------|-------|--------|
| Bubble Sort | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| Selection Sort | O(n²) | O(n²) | O(n²) | O(1) | ❌ |
| Insertion Sort | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| Merge Sort | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ |
| Quick Sort | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ |
