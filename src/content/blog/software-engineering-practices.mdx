## 1. TDD (Test-Driven Development) Thực Chiến
Viết Test (Đề thi) trước, Làm bài (Code) sau. Nếu rớt thì tạch, xanh thì sướng.

**Ví dụ viết hàm Tính Tiền Giỏ Hàng Shopping:**
*(Dùng Jest/JavaScript)*

**Bước 1: RED (Tạch Sấp Mặt vì chưa có Code)**
```javascript
// file: cart.test.js
const { calcCart } = require('./cart');

test('Cộng 2 món hàng có giá 10k ra 20k', () => {
    // Kì vọng: calcCart([{price: 10}, {price:10}]) ==> bằng 20
    expect(calcCart([{price: 10}, {price: 10}])).toBe(20);
});
test('Hàng rỗng thì phí 0đ', () => {
    expect(calcCart([])).toBe(0);
});
```
*Bạn chạy lệnh `npm test` -> Màn hình đỏ chói lọi vả vào mặt vì `calcCart is not defined`.*

**Bước 2: GREEN (Thỏa Mãn Sếp Bằng Mọi Cách Nhanh Nhất)**
```javascript
// file: cart.js
function calcCart(items) {
    if (items.length === 0) return 0;
    return items[0].price + items[1].price; // Code gian lận Hardcode
}
module.exports = { calcCart };
```
*Chạy lại `npm test` -> Xanh Mượt Màn!! (Mừng quá).*

**Bước 3: REFACTOR (Giác Ngộ Rửa Phèn)**
```javascript
// Thay vì hardcode index 0 1, ta code xịn (Quy tắc Refactor không làm hư code đang chạy)
function calcCart(items) {
    return items.reduce((sum, current) => sum + current.price, 0);
}
```
*Chạy lại test -> Vẫn XANH LÈ. Perfect!*

## 2. CI/CD - Gác Kiếm Code, Trông Chờ Đám Mây
Quy trình Tích hợp và Triển khai tự động (Continuous Integration / Continuous Deployment).

**Thảm kịch năm 2010:** Đêm rằm tháng Bảy, SysAdmin ngồi kéo ZIP code lên Xampp/Cpanel. Bị lỗi Typo thiếu dấu chấm phẩy `,` -> Sụp cả App công ty. 2h sáng gọi giật mồng báo lính thức dậy Sửa Trực Tiếp Trên Server.

**Chiêm nghiệm năm 2026 (CI/CD Bằng GitHub Actions):**
File `.github/workflows/deploy.yml`:
```yaml
name: Deploy My App Tự Động Thần Tốc
on:
  push:
    branches: [ "main" ] # Hễ thằng nào Merge vào Nhánh Chính

jobs:
  bui_va_test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node
        uses: actions/setup-node@v3

      - name: Cài Thư Viện
        run: npm ci

      - name: Quét Test Lỗi (TDD Phanh Thỏ)
        run: npm test # Lệnh này bị Đỏ một cái là Bắn Email chửi Devngay, KHÔNG DEPLOY!

      - name: Build Web Production
        run: npm run build

      - name: Triển Khai Lên Vercel/AWS!
        run: npm run deploy:cloud
```
*Mọi việc xảy ra lúc bạn đang uống cà phê sau khi nhấn Nút Mát (Merge) duy nhất.*

## 3. Code Smell - Ngửi Thấy Mùi "Toang"
Biết nhìn Tủ lạnh xem món nào ôi thiu mà dọn. Code thiu là Code Smell.

- Mùi 1: **"God Class" (Lớp Thần Thánh)**. 1 file 5000 dòng. Cái quần gì cũng gom vào (DB, Route, UI). Giải pháp: *SRP* (Single Responsibility).
- Mùi 2: **"Magic Numbers" (Con số ma thuật)**.
  ```typescript
  if (userRole === 3) // Sếp tương tựi: Ủa 3 là giám đốc hay bảo vệ?
  ```
  *Sửa rẹt:* 
  ```typescript
  const ROLE_DIRECTOR = 3; 
  if (userRole === ROLE_DIRECTOR)
  ```
- Mùi 3: **"Long Parameter List" (Hàm tham số dài như sớ táo quân)**.
  ```typescript
  function createUser(name, age, pw, role, id, cmnd, location) {}
  // Rất dễ tráo vị trí tuổi với role -> Sập
  // Sửa liền: gộp vào object
  function createUser({name, age, pw, roleOptions}) {}
  ```

## 4. 12-Factor App - Chân Lý Đưa Trình Lên Cloud Native vĩnh cửu
Một nguyên tắc kinh điển của kỹ sư *Heroku*: "Không nhồi Biến Môi Trường Vô CodeBase".

**❌ Code Quê Mùa (Hardcoded Config):**
```javascript
// config.js
const dbUrl = "mongodb://root:123456@hostvn:27017/shop_db";
const secretKey = "Ams123!@@@!!";
// Đẩy lên Github bị quệt key sụp hệ thống 10 tỷ
```

**✅ Sửa thành Đẳng Cấp Thế Giới (App mây):**
```javascript
// file .env nằm trên ổ cứng (bị Github cấm up lên qua .gitignore)
// DATABASE_URL="mongodb://root..."

// Trong code chắt chiu xài biến môi trường
const dbUrl = process.env.DATABASE_URL;
const secretKey = process.env.JWT_SECRET;
```

Từ đó, 1 mã nguồn Web chạy được ở máy Lập trình (Nhét `.env.local`), Môi trường chạy thử Nháp (Test Staging nhét biến Azure), và Máy thật Production ghim thẻ tín dụng xịn!

## 5. BDD (Behavior-Driven Development) - Lập Trình Theo Kịch Bản Hành Vi
Tiến hóa 1 bậc từ TDD, nếu TDD bốc mùi máy móc kĩ thuật thì **BDD** ép lập trình viên viết Test bằng Tiếng Người (hoặc Tiếng Anh) thuần túy. Business Analyst (BA) đưa kịch bản khách hàng, Dev map thẳng kịch bản đó vào Code Auto-Test! Bạn KHÔNG BAO GIỜ lạc đề!

**Kịch bản BDD (Viết bằng ngôn ngữ Gherkin đưa sếp đọc):**
```gherkin
Feature: Đăng nhập vào hệ thống siêu cấp VVIP

  Scenario: Sếp tổng đăng nhập thành công
    Given (Cho bối cảnh) Tôi đang đứng câm nín ở trang Login `/auth`
    When (Khi Hành động diễn ra) Tôi gõ User là "boss_handsome" và Mật khẩu "anhyeuem123"
    Then (Thì Nhận được kết quả) Hệ thống đá văng tôi vào Giao diện Dashboard Thành Công!
```

**Áp vào Code Test Chạy Thật Trên Cypress (Tự Động Bấm Chuột):**
```javascript
import { Given, When, Then } from 'cypress-cucumber-preprocessor/steps';

// Map 1-1 cực xịn, thằng Code với thằng Sếp/BA lúc này thông não nhau 100%
Given('Tôi đang đứng câm nín ở trang Login {string}', (url) => {
  cy.visit(url);
})

When('Tôi gõ User là {string} và Mật khẩu {string}', (user, pass) => {
  cy.get('#input-username').type(user); // Tự động gõ phím
  cy.get('#input-password').type(pass);
  cy.get('#btn-login').click(); // Bấm bẹp bẹp
})

Then('Hệ thống đá văng tôi vào Giao diện Dashboard Thành Công!', () => {
  cy.url().should('include', '/dashboard!'); // Kiểm tra nhảy trang
})
```

## 6. Refactoring - Phẫu Thuật Thẩm Mỹ Cái Lưới Điện Làng Nhện
Nếu "Code Smell" là cục rác thúi, thì **Refactoring** chính là hành động dọn rác. Nguyên tắc của Lập trình sư xịn: **Đổi cấu trúc vỏ, Không bao giờ đổi Ruột Output**. Chém 1 hàm dài thườn thượt thành 5 hàm nhỏ, nhưng truyền `3` vào vẫn phải nhả ra `5`. Cấm sinh lỗi chênh vênh!

**Kỹ thuật Trấn phái: Guard Clauses (Vệ Binh Gác Cổng / Chém Rụng Tự Trọng Tầng 4)**

**❌ Code Gốc Rợn Người (Arrow Code - Lùi Vào Thành Hình Mũi Tên):**
```typescript
function xetDuyetKhoanVay(khach: KhachHang) {
    if (khach.coMatTaiQuay) { // Tab 1
        if (khach.luong > 10000) { // Tab 2
            if (!khach.dangNoXau) { // Tab 3
                // Não người đã cạn oxy vào tầng thứ 4
                return "Duyệt vay rẹt rẹt tỷ 1!";
            } else {
                throw new Error("Con nợ xấu mà lòi mặt đến đây à?");
            }
        } else {
            throw new Error("Lương bèo bọt, cút!");
        }
    } else {
        throw new Error("Mời ông khách mang thân xác ra quầy!");
    }
}
```

**✅ Chiến Thần Tái Cấu Trúc (Guard Clauses Trút Thác 1 Triều):**
```typescript
function xetDuyetKhoanVay(khach: KhachHang) {
    // 1. Lập chốt chặn Cửa Khẩu. Dính phốt phang lủng sọ văng ra ngay!
    if (!khach.coMatTaiQuay) throw new Error("Mời ông khách mang thân xác ra quầy!");
    if (khach.luong <= 10000) throw new Error("Lương bèo bọt, cút!");
    if (khach.dangNoXau) throw new Error("Con nợ xấu mà lòi mặt đến đây à?");

    // 2. Tinh Anh nào sống sót đi xuống tận khu dòm dòm dưới này thì Núp Trọn Bí Kíp Tiền Tỷ!
    // Lề viền trái của Code Bằng Phẳng Ngay Ngắn vạn dặm!
    return "Duyệt vay rẹt rẹt tỷ 1!";
}
```

## 7. Domain-Driven Design (DDD) - Vị Chúa Của Doanh Nghiệp Cỡ Bự
Phần lớn các web non trẻ (Tiki, Shopee lúc sơ khai) sống mái ở kiểu CSDL, gọi là Dữ Liệu Trung Tâm. Chạy chục năm, logic loạn xà ngầu ngập tràn Controller. Lúc này người ta cứu rỗi bằng **DDD (Thiết Kế Hướng Cốt Lõi Lĩnh Vực)**.

- **Ubiquitous Language (Ngôn Ngữ Chủng Mọi Thống Nhất):** Dân Kinh Doanh nói "Hủy Hợp Đồng", Thằng BA nói "Hủy Hợp Đồng" thì Dưới Lòng Biển Sâu Của Code CŨNG PHẢI tạo hàm `cancelContract()` (Tuyệt đối không để `user.statusUrl = 4`).
- **Rich Domain Model (Lớp Chứa Siêu Quản Trị - Mọng Nước):** Kéo não logic gộp về chính giữa Cục Thực Thể (Entity), không để nó tản mác ra cái đám Service dơ bẩn.

**❌ Dân Coder Mới Lớn Cốt Truyền Thống (Anemic - Cơ Thể Thiếu Máu):**
```typescript
// Thằng Controller ở tầng HTTP nhúng tay đập nát cái Mâm của User
const user = await db.users.findById(1);
if (user.followers >= 10000) {
    user.isKOL = true;      // Nhét dấm nhét dúi cắm cờ KOL khơi khơi bên ngoài
    user.bonusPoints += 50; // Cho luôn tí điểm chơi tết chả ai cấm
}
await db.users.save(user); // Bug đẻ hàng tỷ vì chỗ đéo nào cũng có thể cắm cờ User được!
```

**✅ Đẳng Cấp DDD Máu Tràn (Aggregate Root Quản Quyền Quyết Tử):**
```typescript
// Chấp Cmn Tất Cả Controller Gọi Vào! User tự Tự Trị Chính Mạng Sống Hoành Tráng Của Nó!
class UserRoot {
    private isKOL: boolean = false; // Tủ khóa rập
    private followers: number = 0;

    // Ngôn ngữ doanh nghiệp: Tăng Follow!
    public gainFollower() {
        this.followers++;
        // Nổ check Điều Kiện Vàng (Dấu kĩ kín bưng dưới rốn, đếch ai ngoài biết quy tắc điểm trần 10 Ngàn Mới Tinh Sương Cấp Chậc)
        this.evaluateKOLStatus();
    }

    private evaluateKOLStatus() {
        if (this.followers >= 10000 && !this.isKOL) {
            this.isKOL = true;
            this.bonusPoints += 50;
        }
    }
}

// Giờ Thằng Controller Tủi Thân Chỉ Được Cầu Xin:
const user = await userRepo.findById(1);
user.gainFollower(); // "Ê bạn tăng follow đi, mốc nghìn vạn sao tớ chả biết kệ bạn"
await userRepo.save(user); // Controller vứt bỏ Gánh Nặng Não To, Clean Vô Thiên Lủng!
```
