## 1. Mẫu Kiến Trúc Phổ Biến Nhất: MVC
**(Model - View - Controller)**

Mặc dù có tuổi đời rất lâu, MVC vẫn thống trị bảng xếp hạng vì nó ép Dev chia ra 3 khu vực làm việc riêng rẽ:
- **Thằng Model (Data/Logic):** Mày chỉ lo Query SQL, bắt lỗi data. Mày mù màu, không biết React hay HTML là gì.
- **Thằng View (Giao diện):** Mày chỉ lo vẽ nút bấm đẹp nhất hành tinh. Mày không được dính 1 tí `if(password == "123")` nào.
- **Thằng Controller (Tú bà môi giới):** Mày đứng giữa. Nhận tin báo từ View -> Gọi Model lấy Data -> Trả lại Data cho View vẽ mâm lên bàn.

**Ví dụ Code MVC trong Express Router (Node.js):**
```javascript
// ====== MODEL (models/User.js) ======
const checkLogin = async (usr, pwd) => {
  return await db.query(`SELECT * FROM users WHERE u=? AND p=?`, [usr, pwd]);
}

// ====== CONTROLLER (controllers/AuthController.js) ======
const loginPOST = async (req, res) => {
  const result = await checkLogin(req.body.username, req.body.password); // Gọi Model
  
  if (result) {
    res.render('profilePage', { user: result }); // Nhồi Data vào View
  } else {
    res.render('errorPage', { msg: "Sai Mật Khẩu" });
  }
}

// ====== VIEW (views/profilePage.pug) ======
// <h1> Xin chào bạn hiền #{user.name} </h1>
```

## 2. Clean Architecture (Kiến Trúc Hành Tây Sạch)
Được quảng cáo bằng biểu đồ hình Củ Hành (Onion). Nỗi ám ảnh kinh hoàng của Newbie và là chén cơm của Senior.

**Triết lí TỐI CAO:** "Ông Nền Tảng (Core/Domain) không được phép đụng tới ông Cơ sở hạ tầng (Database/UI)".
Điều này đồng nghĩa **Không được Import Mongoose, TypeORM vào sâu bên trong Core**. Mọi thứ phải thông qua Interface (Dependency Inversion).

**Cấu trúc ví dụ thư mục Clean:**
```text
src/
 ├─ Core/     (KHÔNG PHỤ THUỘC BẤT KỲ CÁI GÌ)
 │   ├─ Entity/User.ts          (Định nghĩa User chuẩn)
 │   └─ UseCase/CreateUserUseCase.ts (Logic tạo User: Validate tên, check tuổi)
 │
 ├─ Infrastructure/ (Phụ thuộc Framework)
 │   ├─ Persistence/MongoUserRepository.ts (Cục bẩn: dính MongoDB)
 │   └─ Http/ExpressController.ts          (Cục bẩn: dính Express/REST)
```

**Code Minh họa Đảo Ngược Phụ Thuộc (DIP) cứu rỗi MongoDB:**
```typescript
// --- LÕI SẠCH BONG (Core) ---
// Định nghĩa sẵn quy tắc bắt hạ tầng phải làm theo
export interface IUserRepository {
  save(user: User): Promise<void>;
}

export class CreateUserUseCase {
  // Bơm Interface vào chứ không dùng Mongoose Model trực tiếp!
  constructor(private userRepo: IUserRepository) {}

  async execute(data: any) {
    if (data.age < 18) throw new Error("Chưa đủ tuổi");
    await this.userRepo.save(new User(data)); // Lưu bất chấp Framework nào!
  }
}

// --- HẠ TẦNG DƠ BẨN (Infrastructure) ---
import { UserModel } from 'mongoose';

export class MongoUserRepository implements IUserRepository {
  async save(user: User) {
    const doc = new UserModel(user); 
    await doc.save(); // Chạm Mongoose Thật ở đây.
  }
}
```
*LỢI ÍCH:* Lúc Sếp bảo "Đổi qua Prisma/PostgreSQL em ơi" -> Mỉm cười cái rẹt, tạo 1 file `PostgresUserRepository.ts`. Phần Core 1000 dòng nghiệp vụ không cần phải sửa 1 dấu chấm phẩy!

## 3. Microservices (Vi Dịch Vụ)
Chia App bự (Monolithic) bằng một Rìu Thần thành nhiều miếng App con.

**Ví dụ thực tế Shopee:**
- Nhánh `user-service`: Code bằng NodeJS + MongoDB. Lo đăng nhập. Lên được 1 Server gánh nổi.
- Nhánh `order-service`: Code bằng Java + PostgreSQL. Đảm bảo giao dịch cực chuẩn. Lên 3 Servers.
- Nhánh `notification-service`: Code bằng Python chạy RabbitMQ, cày tin nhắn rầm rĩ xả rác thả ga.

**Mô hình giao tiếp bắn Event-Driven:**
Thay vì Order gọi điện cho Noti: *"Order kêu nè, gửi Mail cho User hộ tao"*. (Call trực tiếp HTTP rất mất thời gian chờ đợi lỗi mạng sụp theo dây chuyền) -> Microservice dùng Kafka (Pub-Sub).

```javascript
// ORDER SERVICE (Order.js)
// Xong nhiệm vụ, nhổ nước bọt 1 event lên không trung
kafka.publish("ORDER_CREATED_EVENT", { 
    userId: 123, 
    itemName: 'Bàn phím cơ' 
});
// Thằng Order đi uống cafe nghỉ khỏe, không quan tâm việc sau đó.

// NOTIFICATION SERVICE (Mail.js)
// Lúc nào cũng rình rập trên mây
kafka.subscribe("ORDER_CREATED_EVENT", (data) => {
    smtp.send(data.userId, `Cảm ơn bạn đã mua ${data.itemName}`);
});
```

Hệ thống lỏng lẻo cực độ (Decoupled)! Sếp sa thải nguyên team Notification? Vứt mẹ Server nó đi, Thằng Order vẫn nhả Event chạy ầm ầm khách vẫn order ngon ơ. (Tất nhiên khách không nhận được mail thôi).

## 4. Kiến Trúc Nguyên Khối (Monolithic) - Hẩu Lốn Nhưng Mỳ Ăn Liền
Tổ tiên của mọi thể loại hệ thống trước khi đám Microservices bay vô chiếm diễn đàn. **Mọi thứ từ Models, Views, Background Jobs, Mails... trút sạch sành sanh vào chung 1 kho (Repository)** và khởi chạy phịch lên 1 Máy Chủ duy nhất đấu vô 1 DB duy nhất.

**Nỗi đau tột cùng của Monolithic:**
Tính năng Xem Báo Cáo Tính Thuế dở dở ươm ươm ngốn quá nhiều RAM, nó làm Sập Toàn Bộ Ứng Dụng. Dân Đăng Nhập cưng cưng cũng bị văng nốt mặt dù đách liên can.
Muốn đổi cái Mảng Nhỏ chữ Xanh thành chữ Đỏ -> Tắt sạch, Build Cả Một Công Ty lên từ con số 0 Mất 30 phút.

**Nhưng Đừng Vội Che Lạnh! Monolithic là Thượng Đế Nhóm Startup:**
Bạn mở cty chưa rành lãi lỗ, chơi Clean với Microservices tốn 2 tháng xong cháy túi ra đê. Chơi Monolithic nhét hết vô `index.js`, 3 ngày Code ném Vercel đè mặt thị trường. Tiền ập tới rồi, Lúc đó mới Rã Đội Hình Monolithic (Lấy Búa phang vỡ 1 cục Bự thành 10 Cục Nhỏ Microservices).

## 5. Kiến Trúc Xếp Lớp (Layered / N-Tier Architecture) - Ông Tổ Phân Tầng
Kiểu kinh điển 3-Tier của các trường Đại học Nhồi sọ, và sống rất thọ với **Java Spring Boot, .NET MVC**. Bạn đẻ App ra làm 3 Tầng chính:
- **Presentation Layer (Tầng Giao Diện / HTTP):** Người gác cổng. Nhận Request và đẩy JSON Cục Cục lên mặt Khách.
- **Business Logic Layer (Tầng Dịch Vụ - Trái Tim Thiết Bị):** Phán xử cõi Lõi. Lệnh này được duyệt không, tính Toán công thức nợ nần. Thằng này tuyệt đối Mù Về HTML và Mù Về Chỗ Lưu SQL.
- **Data Access Layer (Tầng Cắm Chốt Dữ Liệu - DAL/Repo):** Gần gũi nhất Đất Mẹ (DB). Trách nhiệm Nhồi Ổ Cắm Vào MySQL, Postgres, Firebase... lấy Data xách lên.

**Rule Tối Cao Của Dân Cán Sự Xếp Lớp:** Dòng đời chỉ được gọi xuôi (X->Y->Z). N-Tier Nghiêm cấm mọi hành vi bay cóc! **Presentation cấm Gọi Phọt Vào Data Access Lấy Rổ Database ra dùng! Phải xin Phép Dạ Thưa với Thằng Business**. Do đó tách bạch và tháo dỡ rất dễ thở.

## 6. Serverless Architecture (Kiến Trúc Vô Mặt Máy Chủ)
Cái tên "Vô Máy Chủ" xạo lìn thôi, thực ra Máy Chủ vẫn ở bên Vercel/Amazon, nhưng cái chính là **BẠN KHÔNG CẦN CHĂM SÓC MÁY CHỦ!** Không có vụ cài cắm HDH Ubuntu Linux, Update Nginx, Chống Hack Tỉ Tỉ thứ.

Thay vì Cắm Máy Chạy Ròng Rã 24/7 chờ rác cắn Trả Tiền Cả Tuần, **Serverless đi theo Concept FaaS (Function as a Service)**. Cứ khách Vô Request -> Server bật đinh thức giấc -> Chạy Code Hàm lấy dũ liệu Đưa khách -> Tắt Hàm Phụt Ngủ Luôn.

**Nó Cứu Sinh Ra Sao?**
Trưa Hè, Mạng Nắng Kém 1 Người ghé web $\to$ 1 Hàm Thức, Tốn 0.001Đ. Bất thình lình Độ Mixi Livestream Kéo 10 Vạn Anh Em Ùa Vào Nút Checkout Giờ Vàng $\to$ Serverless Tự Rút Nhân Đôi Phân Bào Thành Đám Đám Mây Mù Khổng Lồ **10,000 cái Hàm Chạy Tuyến Tính Cầm Cự Cùng 1 Thời Khắc**. Khách đi Xong -> Ngủ hết. Tiết kiệm Cực Kì Đã và Cân Khả năng Scale Mượt Hư Không Cõi Biên!.

## 7. MVVM (Model - View - ViewModel) - Bậc Thầy Bắt Trói (Data Binding)
Nếu đám Server cõi Âm thờ phụng MVC thì Cõi Frontend Ngự Trị Bởi Ánh Sáng **MVVM**. Sự Thâm Hậu Tuyệt Đối Nhờ **Two Way Data-Binding Tính Hai Chiều**.

Thằng **ViewModel** Gánh Vác vai trò Khai Hoang Liên Kết (Observer Ngầm Đè):
Bạn Đổi Data Cốt Lõi (Đổi Cặp Chân Mày) -> Mặt Ngoài Browser (View) Lăn Réo Cập Nhật ngay phút mốt!
Bạn Quẹo Gõ Vào Box Tự Nhập Box Input Ngoài Trình Duyệt -> Database Tự Biến Mất Cặp Mắt Theo Ngay Giây Ấy.

**Code Vue/Svelte Dần Chì MVVM Thần Tổ:**
```html
<template>
  <!-- User Quét Phím Kêu "Tùng" Vô Box -->
  <input v-model="hoVaTen"> 
  <!-- Trên Băng Rôn Cổng Chào Tự Hiện Réo Đích Danh "Chào Tùng" (Chả Cần document.getElement Cơm) -->
  <h1>Hí anh em: {{ hoVaTen }}</h1>
</template>

<script>
// Lớp Điệp Viên ViewModel Siêu Viết Lách Móc Đáy
export default {
  data() { return { hoVaTen: "Nam Xoăn" } } // Trụ Cốt Data ở Đây 
}
</script>
```

## 8. Event-Driven & CQRS (Đỉnh Cấp Chịu Tải Siêu Xa)
Lắc Não Lạ Tại sao Shopee Gõ Mua FlashSale Cọc Tiền Nó Lắc Đầu Lag Kinh Tiếng, Mà Nhìn Cái Hình Vuốt List Sản Phẩm Nó Nuột Nà Nhơn Nhớt Vô Tận? Bí Chiêu CQRS Thủng Móng Làng Gióng **(Command Query Responsibility Segregation - Quyết Liệt Ly Biệt Luồng Bơm Cập Nhật Và Luồng Móc Đọc Khí Tượng)**.

Thay Vì 1 Database Server Chịu Cảnh "Vừa Tụt Quần Rặn Cớt - Vừa Đưa Tay Ăn Cơm". Tách Nó Làm 2 Mảng Độc Đáo Khác Vị Rõ Ràng:
- Cơ Chế **C (Command)**: Chuyên Môn Giải Quyết POST, PUT, DELETE (Phạt Viết, Ghi Nặng nề Vào Relational Postgres Cứng Cựa Rì Rì).
- Cơ Chế **Q (Query)**: Giải Phẫu Đọc Bắn GET Trào Máu Lên. Rót Sang ElasticSearch Hoặc Redis Trong 0.01 Giây. Lướt mượt.

Cái Hay Của Vứt Đổi 2 Bên Liên Hệ Rằng Gói Hàng? Chình Là Nhả Lên Trụ Cổng **Event-Driven!**

**Lối Luân Chuyển Phái Hợp Lư Luyện Đan (RabbitMQ/Kafka):**
```javascript
// ====== SERVER GHI SỔ CỜ LÊ (COMMANDS) ======
// Đặt Cọ Chốt Đơn. Tín hiệu Nhẹ Hều Đẩy Tọt 1 Sự Kiện "BỤP_HÀNG_XONG" lên Thung Lũng Tín Gọi Xa 
kafka.publish("ORDER_CREATED", { userId: 5, tK: 'Áo thun' });
return res.status("Mua Ngon"); // Đít Quay Về Cung Cấp Liền, Thách Lũ Loading Chờ Chết Đi.

// ====== Đội MƯỚN WORKER ĐỌC GIẤY BIÊN TRANH (QUERIES & NOTI) ======
// Rải Đàn Chạy Ầm Ầm Theo Sự Nhá Bóng Tín Hiệu. Song Song 1 Nghìn Thằng Giật Sét! Lỗi Đứt Chả Ảnh Hưởng Kênh Chính.
kafka.subscribe("ORDER_CREATED", (data) => {
    // Luồng Phụ Tá 1: Rẽ Nhánh Update Lưu Đọc Database Trắng Sạch Elasticsearch Cho Lũ Trẻ 
    elasticDb.saveViewDocFast(data);
    // Luồng Phụ Tá 2: Email Thông Báo 
    smtp.send(data.userId, `Bác Bán Thành Công Quả ${data.tK} Nhá!`);
});
```

Hệ Lụy Phá Vỡ? Nếu Lũ Notification Gáy Server Lỗi 500 Giãy Chết, Không Cho Mail Về -> Mặt Kệ! Con Web Chi Tiền Thanh Toán User Checkout Nó Vẫn Hoạt Động Xịn Xò Không Xước Móng! Ngăn Chặn "SỤP LÀ SỤP ĐÁM SỤP THEO"!
